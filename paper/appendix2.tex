\chapter{Archived Approaches and Project Evolution}
\label{ch:appendix_archive}

This appendix records the main branches that were explored seriously during the project but did not become part of the final reported pipeline. They are preserved in the repository under \path{ARCHIVE}, and can also be browsed on GitHub at \url{https://github.com/RohanDevereux/Bridging/tree/main/ARCHIVE}. The point of this appendix is not to present them as polished alternatives. The point is to show how the final thesis question became narrower and more defensible.

\section{Correction learning around fast affinity baselines}

One early project idea was to avoid predicting binding affinity entirely from scratch. Instead, the model could start from a fast existing estimator and learn a correction term on top of it. This is an attractive strategy because it combines a hand-designed or physics-inspired baseline with a statistical model that may learn where the baseline fails. Two branches were explored in this spirit: a contact-based baseline using PRODIGY \citep{Vangone2015Contacts} and an endpoint free-energy baseline using MM/GBSA \citep{Genheden2015MMGBSA}.

\subsection{PRODIGY}

The PRODIGY-related code is preserved under \path{ARCHIVE/PRODIGY}. This branch contains dataset parsing, chain-group normalization, estimate prefetching, and a runner that calls an external PRODIGY executable and parses the predicted $\Delta G$. The motivation was straightforward: if a contact-based structural predictor already gives a rough estimate, perhaps a second model can learn the residual error more easily than predicting affinity from raw structural input alone. That idea is reasonable, and it is grounded in the literature behind PRODIGY itself \citep{Vangone2015Contacts}.

In practice, this branch never became a strong thesis backbone. The preserved outputs in the repository are exploratory rather than confirmatory, and the branch was operationally awkward because many complexes failed during execution. More importantly, the baseline itself was not strong enough to make correction learning an attractive central story. For the final thesis, a direct comparison between static and MD-derived graph representations was easier to justify and easier to audit.

\subsection{MM/GBSA}

The MM/GBSA branch is preserved under \path{ARCHIVE/MMGBSA}. It includes AmberTools-based input preparation, HPC launchers, prefetched estimate handling, and a runner that can execute MM/GBSA from a structure or trajectory. The motivation was again correction learning, but with a stronger physics flavour. MM/GBSA occupies a useful middle ground in the literature: it is more physics-based than a purely empirical scorer, but much cheaper than rigorous free-energy calculation \citep{Genheden2015MMGBSA}.

This branch was substantial from an engineering point of view, but it never matured into a clean final result set comparable to the graph-based matrix reported in the main body of the thesis. That mattered. By the time the project direction settled, the final thesis needed a central result that was easy to inspect, rerun, and explain from the repository state. MM/GBSA did not meet that bar, and the approximation-dependence of MM/GBSA itself made it a difficult foundation for the final story \citep{Genheden2015MMGBSA}.

\section{The C$\alpha$ contact-map branch}

The oldest major modelling branch in the repository is the contact-map pipeline preserved under \path{ARCHIVE/CA_contact_map}. This branch converts selected interface residues into fixed two-dimensional tensors built from C$\alpha$--C$\alpha$ contact and distance channels, then trains a convolutional variational autoencoder and downstream regression model on those tensors.

The motivation was straightforward. A contact map gives a fixed-size, image-like representation that is much easier to batch and process than a variable-size protein graph. That design choice has close relatives in protein--ligand learning, where voxelized or tensorized structural inputs have been used successfully \citep{Stepniewska2018Pafnucy}. For an early-stage project, this was a sensible way to get a learnable structural representation running quickly.

The reason this branch was eventually archived is also straightforward. Once the graph pipeline existed, the contact-map representation started to look too compressed. It discarded most residue identity information, side-chain detail, and explicit graph topology, while also depending heavily on patch selection and ordering conventions. The residue-graph formulation retained more structural information and connected more naturally to the MD-derived features that became central to the final project.

\section{The PPB-inspired augmentation and spatiotemporal branch}

The PPB-inspired deep-learning branch is preserved under \path{ARCHIVE/PPB_Affinity_augmentation}. It contains frame-level training, trajectory-level training, and an ensemble that combines predictions from several learned views, together with a substantial vendored model stack under \path{ARCHIVE/PPB_Affinity_augmentation/src/BaselineModel}.

The motivation here was different from the correction-learning branches. This path tried to use MD more directly by learning from frames and trajectories rather than reducing the trajectory to a small set of handcrafted graph features. That direction was reasonable. PPB-Affinity was explicitly introduced as a machine-learning resource for protein drug discovery \citep{Liu2024PPBAffinity}, and recent protein--ligand work has shown that spatio-temporal learning on MD-derived data can be effective in some settings \citep{Libouban2025Spatiotemporal}.

This branch was archived mainly because the project focus narrowed. It introduced a large amount of modelling machinery at once: frame sampling, temporal modelling, augmentation policy, ensembling, and a sizeable external code stack. The later MD-to-graph pipeline was narrower, easier to audit, and easier to connect to the final thesis questions. So this branch is better understood as a serious exploration that was superseded by a more focused direction, not as a simple failed experiment.

\section{Exploratory dataset curation utilities}

The final archived workstream is \path{ARCHIVE/Data_curation}. These scripts cover earlier filtering decisions, partner-size prefetching, subset selection, diverse-sample construction, and older cleaning steps for intermediate datasets.

This material matters because the final dataset definition was not chosen in a vacuum. Earlier in the project there was a real need to balance biological scope, computational tractability, and representation design. Once the final broad, pair-unique PPB-Affinity-derived dataset was fixed, these utilities were no longer part of the active path, but they remain useful as provenance.

\section{What the archived branches contributed}

Although these branches did not survive into the final reported pipeline, they influenced the final thesis in concrete ways. The correction-learning branches clarified that fast structure-only or endpoint baselines were worth comparing against, but were not enough to anchor the whole project. The contact-map branch showed that a fixed tensor representation made the engineering easier at the cost of too much structural compression. The PPB-inspired augmentation branch showed that direct frame- and trajectory-level learning was possible, but that it would be difficult to keep the final thesis as focused and auditable as it needed to be. The data-curation utilities helped narrow the problem to a dataset definition broad enough to be interesting and stable enough to support repeated evaluation.

In that sense, the archived material is not just dead code. It is the record of how the final thesis question became sharper. The thesis eventually settled on a narrower but more defensible comparison: static versus MD-derived residue-graph features on a matched protein--protein affinity dataset, evaluated under repeated held-out resampling.
