\chapter{Repository and Implementation Availability}
\label{ch:code}

The implementation for this thesis is available in the public repository
\url{https://github.com/RohanDevereux/Bridging}.
Rather than reproducing large blocks of source code in the thesis itself, this chapter points to the parts of the repository that correspond directly to the pipeline described in the main text.

The active code path used for the final results is concentrated in a small part of the repository. Molecular-dynamics preparation and simulation live in \path{src/bridging/MD} and can also be browsed directly on GitHub at \url{https://github.com/RohanDevereux/Bridging/tree/main/src/bridging/MD}. Graph preparation, including DeepRank2 integration, dynamic feature extraction, force-proxy construction, and graph-view materialization, lives in \path{src/bridging/graphvae/prep} and \path{src/bridging/graphvae/runners}, corresponding to \url{https://github.com/RohanDevereux/Bridging/tree/main/src/bridging/graphvae/prep} and \url{https://github.com/RohanDevereux/Bridging/tree/main/src/bridging/graphvae/runners}. Model definitions, dataset preprocessing, the supervised baseline, and the latent-space regression code live in \path{src/bridging/graphvae/ml}, corresponding to \url{https://github.com/RohanDevereux/Bridging/tree/main/src/bridging/graphvae/ml}. Shared parsing and affinity-conversion utilities live in \path{src/bridging/utils}.

The key committed artifacts used in the final thesis are also present in the repository snapshot. The curated dataset used by the active pipeline is \path{src/bridging/processedData/PPB_Affinity_broad_pairuniq_train80_test20.csv}. The matched full and interface graph-record files used in the final confirmatory resampling study are \path{graphvae_force_v3_results/src/bridging/generatedData/graphvae/PPB_Affinity_broad_pairuniq_train80_test20_views_force_mdiface_v3/md_closest_pair_patch/graph_records_full.pt} and \path{graphvae_force_v3_results/src/bridging/generatedData/graphvae/PPB_Affinity_broad_pairuniq_train80_test20_views_force_mdiface_v3/md_closest_pair_patch/graph_records_interface.pt}. The compact summary of the final 52-configuration resample matrix is \path{RESULTS/final_resample_matrix_summary.csv}. Chapters~\ref{ch:results} and~\ref{ch:discussion} are written against these exact artifacts.

The repository also contains HPC launch scripts under \path{hpc}. These scripts are useful for rerunning the larger stages of the pipeline, but they are wrappers around the Python entry points rather than a separate source of logic. For understanding or auditing the implementation, the Python modules listed above are the important locations.

Finally, the repository contains an \path{ARCHIVE} directory, also visible at \url{https://github.com/RohanDevereux/Bridging/tree/main/ARCHIVE}. This material is intentionally preserved because it records substantial work that shaped the final direction of the project, even when that work did not end up in the final reported pipeline. Appendix~\ref{ch:appendix_archive} explains those archived branches, their motivations, and why they were ultimately not part of the final thesis contribution.
