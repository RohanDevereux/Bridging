\chapter{Introduction}
\label{ch:intro}

Protein--protein binding affinity prediction asks for a quantitative map from structure to binding free energy. The problem is important in molecular recognition and protein design, and it is difficult because the target is thermodynamic while the input is usually a single experimental structure \citep{Modell2016PPIs,Guo2022PPBAReview,Liu2024PPBAffinity}. This thesis studies that problem on a curated subset of PPB-Affinity \citep{Liu2024PPBAffinity}.

The thesis asks whether a short explicit-solvent molecular-dynamics simulation provides useful predictive information beyond a static residue graph. It evaluates one implemented residue-graph pipeline. Each complex is prepared for molecular dynamics, simulated in OpenMM, converted to a residue graph with DeepRank2, augmented with trajectory-derived summaries, and then evaluated with either a masked GraphVAE or a direct supervised graph regressor. The processed broad table contains 2818 complexes, but the final comparison uses a smaller compute-bounded 1437-complex analysis subset for which paired full and interface-restricted graph views are available.

The implementation, curated inputs, and compact result summaries are available in the repository \url{https://github.com/RohanDevereux/Bridging}. Appendix~\ref{ch:appendix_archive} records the archived branches explored before the final pipeline was fixed.

The thesis asks four empirical questions. Is a full residue graph better than an interface-restricted representation? Do the molecular-dynamics-derived summaries improve prediction over static graph features alone? Does a masked GraphVAE yield a better predictive representation than a direct supervised graph regressor with the same encoder backbone? How stable are the answers under repeated held-out evaluation rather than a single train/test split?

Chapter~\ref{ch:litreview} places the work in the literature on protein--protein affinity prediction, molecular simulation, and learned structural representations. Chapter~\ref{chap:methods} describes the implemented pipeline. Chapter~\ref{ch:results} reports the final model matrix. Chapter~\ref{ch:discussion} gives the main conclusions. The appendices cover archived branches, the complete 52-model matrix, and the repository layout.
